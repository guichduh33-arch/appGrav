## Notes for Implementation

1. **i18n Keys Structure**: Use dot notation for nested keys
   - Example: `purchasing.suppliers.title` → "Suppliers"
   - Example: `purchasing.orders.status.draft` → "Draft"

2. **Pluralization**: Consider singular/plural forms
   - `supplier` (1) vs `suppliers` (multiple)
   - `order` (1) vs `orders` (multiple)

3. **Currency & Units**: Keep locale-specific formatting
   - EUR symbol: €
   - Date format: Use i18n date formatting

4. **Validation Messages**: Include field names in error messages
   - Template: `{field} is required` → "Supplier is required"

5. **Success Messages**: Provide clear feedback
   - "Supplier created successfully"
   - "Purchase order saved"
   - "Item returned successfully"
