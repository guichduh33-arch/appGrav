### States

1. **Idle** - No payments added, all methods available
2. **Adding** - User selecting payment method and entering amount
3. **Validating** - Amount being validated
4. **Complete** - Total reached or exceeded, ready to complete

### Interaction Flow

1. User clicks payment method button
2. For Cash: Numpad appears for amount entry
3. For Others: Amount input with remaining pre-filled
4. User confirms amount â†’ Payment added to list
5. Progress bar updates
6. Repeat until total reached
7. Complete button enabled when total >= order amount
